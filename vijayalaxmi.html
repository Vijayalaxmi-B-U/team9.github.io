REFLECTIONS

Nature & Algorithms: Cycles show iteration, layered growth shows recursion, and ant movement reflects backtracking.

Efficiency: Time and space complexity help choose fast, scalable algorithms. Key growth rates: O(1), O(log n), O(n), O(n²).

Design Principles: Balanced trees, DFS/BFS strategies, and sorting techniques helped understand how structure improves efficiency.

Hierarchical Structures: Trees like BST, AVL, Red-Black, Heaps, and Tries organize data efficiently for searching, sorting, and indexing.

Array Queries: Segment Trees, BIT, and lookup tables speed up repeated range queries and updates on large datasets.

Trees vs Graphs: Trees represent hierarchy; graphs model networks. Traversed using DFS, BFS, and used in systems like GPS.

Sorting & Searching: Algorithms like Merge/Quick Sort and Binary Search connect to real tasks such as ranking, filtering, and searching.

Graph Algorithms: MST saves cost in networks; shortest-path (Dijkstra) powers maps, routing, and logistics.

Design Techniques: Backtracking, divide-and-conquer, brute force, string matching, and shortest-path algorithms solve a wide range of problems.

Tourist Info & Navigation – Algorithms

Dijkstra – Shortest Path

Calculates the shortest route between tourist locations, reducing travel time and improving navigation efficiency.

ALGORITHM Dijkstra(G, s) Initialize(Q) for each v in V dv ← ∞ pv ← null Insert(Q, v, dv) ds ← 0 Decrease(Q, s, ds) VT ← ∅ for i ← 0 to |V| − 1 u* ← DeleteMin(Q) VT ← VT ∪ {u*} for each u adjacent to u* if du* + w(u*, u) < du du ← du* + w(u*, u) pu ← u* Decrease(Q, u, du)

BFS & DFS – Exploration

BFS explores nearby attractions level-by-level, while DFS checks deep connectivity between locations.

BFS(G, s) Mark s visited Enqueue s while queue not empty u ← dequeue for each v adjacent to u if not visited mark visited enqueue v DFS(G, s) Mark s visited for each v adjacent to s if not visited DFS(G, v)

Trie & KMP – Fast Search

Trie enables prefix search of tourist places and KMP helps fast keyword matching.

Trie Insert(word) for each character c in word if node[c] not exist create node[c] move to node[c] mark end KMP(text, pattern) Compute LPS array Pattern matching using LPS

Greedy – Crowd Control

Chooses the least crowded tourist spot to reduce congestion.

Select place with minimum crowd value Recommend to tourists

Dynamic Programming – Tour Planning

Helps plan optimal tourist visits within limited time.

DP[i][t] = maximum satisfaction DP[i][t] = max(DP[i-1][t], value[i] + DP[i-1][t - time[i]])

ACCOMMODATION

Accommodation is a core pillar of tourism. Comfortable, safe, and eco-friendly stays improve tourist satisfaction, extend visit duration, and boost the local economy. Smart accommodation systems reduce booking conflicts, ensure accessibility, and promote sustainable resource usage in smart cities.

Hashing – Smart Booking System

Hashing enables instant room availability checks and prevents double bookings by mapping room IDs to their current status.

Insert(roomID, status) into HashTable Search(roomID) if status == available confirm booking else show alternative rooms

Union-Find – Room & Building Management

Union-Find groups connected rooms or hotel blocks, helping manage large accommodations and ensuring efficient allocation during peak demand.

MakeSet(room) Union(roomA, roomB) Find(room) Check if rooms belong to same block

Heap – Best Room Selection

Heaps prioritize rooms based on price, rating, or availability, allowing tourists to quickly select the most suitable option.

Insert(room, priority) ExtractMin() // lowest price or best rating

Sliding Window – Demand Analysis

Sliding Window analyzes booking trends over fixed time intervals to predict peak demand and optimize pricing strategies.

windowSum = sum(bookings[0..k-1]) for i = k to n windowSum += bookings[i] windowSum -= bookings[i-k]

Dynamic Programming – Optimal Stay Planning

Dynamic Programming helps tourists choose the best accommodation within a fixed budget while maximizing comfort and stay duration.

DP[i][b] = maximum comfort DP[i][b] = max(DP[i-1][b], comfort[i] + DP[i-1][b - cost[i]])
